      <p><b>Microsoft Developer Network</b> (<b>MSDN</b>) is the portion of <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> responsible for managing the firm's relationship with developers and testers, such as hardware developers interested in the <a href="/wiki/Operating_system" title="Operating system">operating system</a> (OS), and software developers developing on the various OS platforms or using the <a href="/wiki/Application_programming_interface" title="Application programming interface">API</a> or scripting languages of Microsoft's applications. The relationship management is situated in assorted media: <a href="/wiki/Web_site" class="mw-redirect" title="Web site">web sites</a>, <a href="/wiki/Newsletter" title="Newsletter">newsletters</a>, <a href="/wiki/Developer_conference" class="mw-redirect" title="Developer conference">developer conferences</a>, trade media, <a href="/wiki/Blog" title="Blog">blogs</a> and <a href="/wiki/DVD" title="DVD">DVD</a> distribution. The life cycle of the relationships ranges from legacy support through evangelizing potential offerings.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (November 2016)">citation needed</span></a></i>]</sup> </p>

      <h2><span class="mw-headline" id="Websites">Websites</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=1" title="Edit section: Websites">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>MSDN's primary web presence at <i>msdn.microsoft.com</i> is a collection of sites for the developer community that provide information, documentation, and discussion that is authored both by Microsoft and by the community at large. Recently, Microsoft has placed emphasis on incorporation of forums, blogs, library annotations and social bookmarking to make MSDN an open dialog with the developer community rather than a one-way service.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> The main website, and most of its constituent applications below are available in 56<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> or more languages. </p> 
      <h3><span class="mw-headline" id="Library">Library</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=2" title="Edit section: Library">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p><b>MSDN Library</b> is a library of official technical documentation content intended for <a href="/wiki/Software_developer" class="mw-redirect" title="Software developer">developers</a> developing for <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>. MSDN Library documents the <a href="/wiki/Application_Programming_Interface" class="mw-redirect" title="Application Programming Interface">APIs</a> that ship with <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> products and also includes sample code, technical articles, and other programming information. It is available free on the web and on <a href="/wiki/Compact_disc" title="Compact disc">CDs</a> and <a href="/wiki/DVD" title="DVD">DVDs</a> for paid MSDN subscribers. Initially, the disc version was only available as part of an <a href="/wiki/MSDN" class="mw-redirect" title="MSDN">MSDN</a> subscription and was released on a quarterly basis (January, April, July and October). However, in recent times (2006 and later), it can be freely downloaded from Microsoft Download Center in form of <a href="/wiki/ISO_image" title="ISO image">ISO images</a><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup><sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup><sup id="cite_ref-MSDN2008SP1_5-0" class="reference"><a href="#cite_note-MSDN2008SP1-5">[5]</a></sup> for CD/DVD releases are no longer published quarterly. Instead, its release schedule is now aligned with major software releases (major <a href="/wiki/Visual_Studio" class="mw-redirect" title="Visual Studio">Visual Studio</a> release, major <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> release or service packs), (up to Visual Studio 2008).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (November 2016)">citation needed</span></a></i>]</sup> </p>
      <p><a href="/wiki/Visual_Studio_Express" class="mw-redirect" title="Visual Studio Express">Visual Studio Express</a> edition integrates only with MSDN Express Library, which is a subset of the full MSDN Library, although either MSDN edition can be freely downloaded and installed standalone. </p>
      <p>In <a href="/wiki/Visual_Studio_2010" class="mw-redirect" title="Visual Studio 2010">Visual Studio 2010</a> MSDN Library is replaced with the new Help System, which is installed as a part of Visual Studio 2010 installation. Help Library Manager is used to install Help Content books covering selected topics. </p>
      <p>In 2016, Microsoft introduced the new technical documentation platform, <a href="/wiki/Microsoft_Docs" title="Microsoft Docs">Microsoft Docs</a>, intended as a replacement of TechNet and MSDN libraries. <sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> <sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> Over the next two years, the content of MSDN Library was gradually migrated into Microsoft Docs. <sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> Now most of MSDN Library pages redirect to the corresponding Microsoft Docs pages. </p> 
      <h4><span class="mw-headline" id="Integration_with_Visual_Studio">Integration with Visual Studio</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=3" title="Edit section: Integration with Visual Studio">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>Each edition of MSDN Library can only be accessed with one help viewer (Microsoft Document Explorer or other help viewer), which is integrated with the <i>then current</i> single version or sometimes two versions of Visual Studio. In addition, each new version of Visual Studio does not integrate with an earlier version of MSDN. A compatible MSDN Library is released with each new version of Visual Studio and included on Visual Studio DVD. As newer versions of Visual Studio are released, newer editions of MSDN Library do not integrate with older Visual Studio versions and do not even include old/obsolete documentation for deprecated or discontinued products. MSDN Library versions can be installed side-by-side, that is, both the older as well as the newer version of MSDN Library can co-exist. </p> 
      <h3><span class="mw-headline" id="Forums">Forums</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=4" title="Edit section: Forums">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>MSDN Forums are the <a href="/wiki/Internet_forum" title="Internet forum">web-based forums</a> used by the community to discuss a wide variety of software development topics. MSDN Forums were migrated to an all-new platform during 2008 that provided new features designed to improve efficiency such as inline preview of threads, <a href="/wiki/AJAX" class="mw-redirect" title="AJAX">AJAX</a> filtering, and a slide-up post editor. </p> 
      <h3><span class="mw-headline" id="Blogs">Blogs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=5" title="Edit section: Blogs">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>MSDN blogs is a series of <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> <a href="/wiki/Blog" title="Blog">blogs</a> hosted under <code>blogs.msdn.com</code> <a href="/wiki/Domain_name" title="Domain name">domain name</a>. Some blogs are dedicated to a product – e.g. <a href="/wiki/Visual_Studio" class="mw-redirect" title="Visual Studio">Visual Studio</a>,<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>,<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> <a href="/wiki/PowerShell" title="PowerShell">PowerShell</a><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup> – or a version of a product – e.g. <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a>,<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup> <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a><sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup> – while others belong to a Microsoft employee, e.g. <a href="/wiki/Michael_Howard_(Microsoft)" title="Michael Howard (Microsoft)">Michael Howard</a><sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[14]</a></sup> or Raymond Chen.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[15]</a></sup> </p> 
      <h3><span class="mw-headline" id="Social_bookmarking">Social bookmarking</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=6" title="Edit section: Social bookmarking">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p><a href="/wiki/Social_bookmarking" title="Social bookmarking">Social bookmarking</a> on MSDN Social was first launched in 2008, built on a new web platform that has <a href="/wiki/Tag_(metadata)" title="Tag (metadata)">user-tagging</a> and <a href="/wiki/Web_feeds" class="mw-redirect" title="Web feeds">feeds</a> at its core. The goal of the social bookmarking application is to provide a method whereby members of the developer community can: </p> 
      <ul>
       <li>Contribute to a database of quality links on any topic from across the web. By filtering on one or more tags, (e.g. ".net" and "database") users can discover popular or recent links and subscribe to a feed of those links.</li> 
       <li>Find and follow experts' recommended sites. Each profile page includes a feed of the user's contributions. Users can be discovered through a drop-down menu on each bookmark.</li> 
       <li>Demonstrate their expertise through the links displayed in their profile.</li> 
       <li>Store their favorite links online.</li>
      </ul> 
      <p>The initial release of the application provides standard features for the genre, including a <a href="/wiki/Bookmarklet" title="Bookmarklet">bookmarklet</a> and import capabilities. The MSDN web site is also starting to incorporate feeds of social bookmarks from experts and the community, displayed alongside feeds from relevant bloggers.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">[16]</a></sup> </p>
      <p>Social Bookmarks was discontinued on October 1, 2009. </p> 
      <h3><span class="mw-headline" id="Gallery">Gallery</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=7" title="Edit section: Gallery">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>MSDN Gallery is a repository of community-authored code samples and projects. Launched in 2008, the purpose of the site is still evolving to complement <a href="/wiki/Codeplex" class="mw-redirect" title="Codeplex">Codeplex</a>, the <a href="/wiki/Open-source_software" title="Open-source software">open-source</a> project hosting site from <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>. </p> 
      <h2><span class="mw-headline" id="Software_subscriptions">Software subscriptions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=8" title="Edit section: Software subscriptions">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>MSDN has historically offered a subscription package whereby developers have access and licenses to use nearly all Microsoft software that has ever been released to the public. Subscriptions are sold on an annual basis, and cost anywhere from US$1,000 to US$6,000 per year per subscription, as it is offered in several tiers. Holders of such subscriptions (except the lowest library-only levels) receive new Microsoft software on DVDs or via downloads every few weeks or months. The software generally comes on specially marked MSDN discs, but contains the identical retail or volume-license software as it is released to the public. </p>
      <p>Although in most cases the software itself functions exactly like the full product, the MSDN end-user license agreement<sup id="cite_ref-eula_17-0" class="reference"><a href="#cite_note-eula-17">[17]</a></sup> prohibits use of the software in a business production environment. This is a legal restriction, not a technical one. As an example, MSDN regularly includes the latest Windows operating systems (such as <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a>, <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a> and <a href="/wiki/Windows_8.1" title="Windows 8.1">Windows 8.1</a>), server software such as <a href="/wiki/SQL_Server_2008" class="mw-redirect" title="SQL Server 2008">SQL Server 2008</a>, development tools such as <a href="/wiki/Visual_Studio" class="mw-redirect" title="Visual Studio">Visual Studio</a>, and applications like <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> and <a href="/wiki/MapPoint" class="mw-redirect" title="MapPoint">MapPoint</a>. For software that requires a <a href="/wiki/Product_key" title="Product key">product key</a>, a Microsoft website generates these on demand. Such a package provides a single Microsoft enthusiast with access to nearly everything Microsoft offers. However, a business caught with an office full of PCs and servers running the software included in an MSDN subscription without the appropriate non-MSDN licenses for those machines would be treated no differently in a <a href="/wiki/Software_licensing_audit" title="Software licensing audit">software licensing audit</a> than if the software were obtained through piracy. </p>
      <p>Microsoft's MSDN license agreement<sup id="cite_ref-eula_17-1" class="reference"><a href="#cite_note-eula-17">[17]</a></sup> makes a specific exception for <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>, allowing the subscription holder to personally use it for business purposes without needing a separate license — but only with the "MSDN Premium Subscription" and even so only "directly related to the design, development and test and/or documentation of software projects" as stated in the MSDN licensing FAQ. As would be expected, any software created with the development tools (like Visual Studio), along with the runtime components needed to use it, isn't restricted in any way by Microsoft either — such software can and regularly is used for business production purposes. The license agreement refers to several other items in the subscription and grants additional similar exceptions as appropriate. </p>
      <p>An MSDN subscriber is entitled to activate as many copies as needed for his/her own development purposes. Therefore, if a computer enthusiast has 20 computers (actual or <a href="/wiki/Virtual_machine" title="Virtual machine">virtual</a>) for software development (and aren't acting as part of a business, for example, a server farm), one subscription allows all 20 of those computers to be running their own separate copy of Windows, Office, and any other Microsoft product. After a few installations, the activation keys will stop allowing automatic <a href="/wiki/Product_activation" title="Product activation">product activation</a> over the Internet, but after a telephone call to the Product Activation hotline to confirm that the installations are indeed legitimate and consistent with the license agreement, the activations are granted over the phone. </p>
      <p>Even though an MSDN subscription is on an annual basis (for retail subscriptions—volume licensing subscriptions can be multi-year), the license to use the software, according to the agreement,<sup id="cite_ref-eula_17-2" class="reference"><a href="#cite_note-eula-17">[17]</a></sup> does not terminate. However, the individual will no longer be entitled to any upgrades after the subscription has expired. An MSDN subscription also allows access to legacy Microsoft products. Although they aren't included in the regular CD/DVD shipments, subscribers can download older software such as <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> 5.0 and <a href="/wiki/Windows_3.1" class="mw-redirect" title="Windows 3.1">Windows 3.1</a> from MSDN Subscriber Downloads. Such software usually comes in the form of <a href="/wiki/ISO_image" title="ISO image">ISO</a> or floppy disk image files that allow the subscriber to reproduce the original installation media after the download. </p> 
      <h2><span class="mw-headline" id="Information_service">Information service</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=9" title="Edit section: Information service">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The division runs an information service provided by Microsoft for <a href="/wiki/Software_developer" class="mw-redirect" title="Software developer">software developers</a>. Its main focus is on Microsoft's <a href="/wiki/Microsoft_.NET" class="mw-redirect" title="Microsoft .NET">.NET platform</a>, however it also features articles on areas such as programming practices and <a href="/wiki/Design_pattern_(computer_science)" class="mw-redirect" title="Design pattern (computer science)">design patterns</a>. Many resources are available for free online, while others are available by <a href="/wiki/Snail_mail" title="Snail mail">mail</a> via a subscription. </p>
      <p>Depending on subscription level, subscribers may receive early editions of Microsoft <a href="/wiki/Operating_system" title="Operating system">operating systems</a> or other Microsoft products (<a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> applications, <a href="/wiki/Visual_Studio_.NET" class="mw-redirect" title="Visual Studio .NET">Visual Studio</a>, etc.). </p>
      <p>Universities and high schools can enroll in <a href="/wiki/DreamSpark_Premium" class="mw-redirect" title="DreamSpark Premium">DreamSpark Premium</a> program, which provides access to some Microsoft developer software for their computer science and engineering students (and possibly other students or faculty as well). A DreamSpark Premium account cannot be used to access the subscriber's section of the MSDN website or its downloads. </p> 
      <h2><span class="mw-headline" id="MSDN_Magazine"><i>MSDN Magazine</i></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=10" title="Edit section: MSDN Magazine">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Microsoft provides the editorial content for <i>MSDN Magazine</i>, a monthly publication. The magazine was created as a merger between <i>Microsoft Systems Journal</i> (MSJ) and <i>Microsoft Internet Developer</i> (MIND) magazines in March 2000. MSJ back issues are available online.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">[18]</a></sup> <i>MSDN Magazine</i> was available as a print magazine in the United States, and online in 11 languages. The last issue of the magazine is planned for November 2019.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">[19]</a></sup> </p> 
      <h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=11" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>MSDN was launched in September 1992<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">[20]</a></sup> as a quarterly, CD-ROM-based compilation of technical articles, sample code, and software development kits. The first two MSDN CD releases (September 1992 and January 1993) were marked as pre-release discs (P1 and P2, respectively).<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">[21]</a></sup> Disc 3, released in April 1993, was the first full release. In addition to CDs, there was a 16-page tabloid newspaper, <i>Microsoft Developer Network News</i>, edited by <a href="/wiki/Andrew_Himes" title="Andrew Himes">Andrew Himes</a>, who had previously been the founding editor of <a href="/wiki/MacTech" title="MacTech">MacTech</a>, the premiere Macintosh technology journal.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">[22]</a></sup> A Level II subscription was added in 1993, that included the MAPI, ODBC, TAPI and VFW SDKs.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">[23]</a></sup> </p>

      <p>MSDN2 was opened in November 2004 as a source for <a href="/wiki/Visual_Studio" class="mw-redirect" title="Visual Studio">Visual Studio 2005</a> <a href="/wiki/Application_Programming_Interface" class="mw-redirect" title="Application Programming Interface">API</a> information, with noteworthy differences being updated web site code, conforming better to <a href="/wiki/W3C" class="mw-redirect" title="W3C">web standards</a> and thus giving a long-awaited improved support for alternative <a href="/wiki/Web_browser" title="Web browser">web browsers</a> to <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> in the API browser. In 2008, the original MSDN cluster was retired and MSDN2 became msdn.microsoft.com.<sup id="cite_ref-24" class="reference"><a href="#cite_note-24">[24]</a></sup> </p> 
      <h3><span class="mw-headline" id="Dr_GUI_and_the_MSDN_Writers_Team">Dr GUI and the MSDN Writers Team</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Developer_Network&amp;action=edit&amp;section=12" title="Edit section: Dr GUI and the MSDN Writers Team">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>In 1996, <a href="/w/index.php?title=Bob_Gunderson&amp;action=edit&amp;redlink=1" class="new" title="Bob Gunderson (page does not exist)">Bob Gunderson</a> began writing a column in <i>Microsoft Developer Network News</i>, edited by <a href="/wiki/Andrew_Himes" title="Andrew Himes">Andrew Himes</a>, using the pseudonym "Dr.GUI". The column provided answers to questions submitted by MSDN subscribers. The caricature of Dr. GUI was based on a photo of Gunderson. When he left the MSDN team, <a href="/w/index.php?title=Dennis_Crain&amp;action=edit&amp;redlink=1" class="new" title="Dennis Crain (page does not exist)">Dennis Crain</a> took over the Dr. GUI role and added medical humor to the column. Upon his departure, Dr. GUI became the composite identity of the original group (most notably Paul Johns) of Developer Technology Engineers that provided in-depth technical articles to the Library. The early members included: Bob Gunderson, <a href="/w/index.php?title=Dale_Rogerson&amp;action=edit&amp;redlink=1" class="new" title="Dale Rogerson (page does not exist)">Dale Rogerson</a>, <a href="/w/index.php?title=R%C3%BCdiger_R._Asche&amp;action=edit&amp;redlink=1" class="new" title="Rüdiger R. Asche (page does not exist)">Rüdiger R. Asche</a>, <a href="/w/index.php?title=Ken_Lassesen&amp;action=edit&amp;redlink=1" class="new" title="Ken Lassesen (page does not exist)">Ken Lassesen</a>, Nigel Thompson (a.k.a. Herman Rodent), <a href="/w/index.php?title=Nancy_Cluts&amp;action=edit&amp;redlink=1" class="new" title="Nancy Cluts (page does not exist)">Nancy Cluts</a>, Paul Johns, Dennis Crain, and <a href="/w/index.php?title=Ken_Bergmann&amp;action=edit&amp;redlink=1" class="new" title="Ken Bergmann (page does not exist)">Ken Bergmann</a>. Nigel Thompson was the development manager for Windows Multimedia Extensions that originally added multimedia capabilities to Windows. Renan Jeffreis produced the original system (Panda) to publish MSDN on the Internet and in HTML instead of the earlier multimedia viewer engine. Dale Rogerson, Nigel Thompson and Nancy Cluts all published MS Press books while on the MSDN team. As of August 2010, only Dennis Crain and Dale Rogerson remain employed by Microsoft. </p>
